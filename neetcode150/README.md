# NeetCode 150 - Complete Interview Preparation

A comprehensive collection of 150 curated LeetCode problems organized by pattern, designed for mastering coding interviews.

## Why NeetCode 150?

The NeetCode 150 is one of the most popular coding interview preparation lists because:
- **Pattern-Based Learning**: Problems grouped by algorithmic patterns, not just data structures
- **Video Explanations**: Each problem has a detailed video walkthrough
- **Interview Focused**: Curated based on actual FAANG interview frequency
- **Progressive Difficulty**: Build skills incrementally within each category
- **Comprehensive Coverage**: Covers all major patterns and techniques

## Key Patterns to Master

1. **Arrays & Hashing**: Hash maps for O(1) lookups, frequency counting
2. **Two Pointers**: Converging pointers, same direction pointers
3. **Sliding Window**: Variable/fixed size windows for subarrays
4. **Stack**: Monotonic stack, bracket matching, function call simulation
5. **Binary Search**: Search space reduction, answer binary search
6. **Linked List**: Fast/slow pointers, dummy nodes, reversal
7. **Trees**: DFS (pre/in/post-order), BFS (level-order), BST properties
8. **Tries**: Prefix matching, word search
9. **Heap**: Top K problems, merge K, median tracking
10. **Backtracking**: Decision tree, pruning, state restoration
11. **Graphs**: DFS, BFS, cycle detection, topological sort
12. **Advanced Graphs**: MST (Kruskal/Prim), Dijkstra, Union Find
13. **Dynamic Programming**: State definition, recurrence relation, base cases
14. **Greedy**: Local optimal â†’ global optimal, proof of correctness
15. **Intervals**: Sorting, merging, overlapping detection
16. **Bit Manipulation**: XOR properties, bit masks, powers of 2

## Study Tips

1. **Watch Videos First**: NeetCode provides excellent explanations - watch before coding
2. **Understand, Don't Memorize**: Focus on why the solution works
3. **Pattern Recognition**: After solving, identify the pattern used
4. **Spaced Repetition**: Revisit problems after 1 day, 1 week, 1 month
5. **Time Yourself**: Practice under interview conditions (45 min per problem)
6. **Explain Out Loud**: Verbalize your thought process
7. **Multiple Solutions**: Try brute force â†’ optimized â†’ optimal
8. **Edge Cases**: Empty input, single element, duplicates, negatives
